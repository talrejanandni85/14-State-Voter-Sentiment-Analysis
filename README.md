## 🔍 Project Overview

A political research organization runs an ongoing online panel of voters across 14 battleground states, surveying them on policy questions using a **non-binary 0–100 allocation scale** — capturing the *strength* of an opinion, not just its direction.

Each report cycle, raw survey exports need to be processed into weighted demographic breakouts and state-level distributions for client-facing reports covering multiple policy topics. I built and maintained the R pipeline that takes raw panel exports and produces the structured outputs used in every report.

This repo shows the pipeline for one report series. A second parallel series covers a different policy area and uses the same codebase.

---

## ❓ The Analytical Problem

- Raw exports come as long-format files: one row per respondent per question, with separate demographic files linked by User ID
- Demographic data (income, education, area) arrives in a separate file and must be merged and pivoted
- All results must be **weighted** using respondent weights to correct for panel imbalances
- Outputs need to be broken out by: gender, party, race, BIPOC aggregate, age group, income group, education group — both overall and by state
- Distribution of opinion must be bucketed into five 20-point bands (0–19, 20–39, 40–59, 60–79, 80–100) and calculated as weighted percentages
- Small subgroups (n < 30) must be flagged for suppression before delivery
- Final outputs are Excel files used directly in PowerPoint client reports

---

## 🛠️ What I Built

### `analysis/baseline_survey_analysis.Rmd` — Core Processing Pipeline
- Loads and merges raw survey responses with demographic supplement via User ID
- Includes merge diagnostics (match rate, unmatched counts) for QC
- Deduplicates to one response per respondent using the most recent submission date
- Filters to 14 target states
- Pivots demographic file from long to wide; creates `IncomeGroup` (under/over $75k) and `EducationGroup` (college/non-college) binary variables
- Calculates **weighted mean response value** for 13 demographic categories using `weighted.mean()` with respondent weights
- Calculates **BIPOC aggregate** (all non-White races combined)
- Buckets responses into five distribution bands and calculates weighted frequency percentages, both overall and by state (via loop)
- Flags subgroups below n=30 threshold
- Exports all results to a multi-sheet Excel file

### Output files (in `outputs/`)
Generated by the Rmd, not committed to the repo:
- `policy_a_analysis_results.xlsx` — demographic breakouts + distribution tables
- `policy_b_analysis_results.xlsx` — same structure for second question

## 📊 Key Analytical Findings (Example Report)

**Policy Question A (Support/Oppose):**
- Overall weighted average: **49.9 / 100** — near the midpoint, indicating a divided electorate
- Partisan divide: Republicans **84.7** vs. Democrats **24.6** — a 60-point gap
- Distribution is bimodal: 34.9% scored 0–19, 38.5% scored 80–100 — highly polarized, little middle ground
- State range: Florida highest at **60.8**, Oregon lowest at **42.1**

**Policy Question B (Coalition Statement):**
- Overall weighted average: **70.6 / 100** — strong net support
- Partisan pattern inverts: Democrats **85.0** vs. Republicans **57.9**
- Racial minorities score higher (Black/African American: 78.5, Native American: 78.2)
- Women outpace men (75.8 vs. 71.0)

## ⚙️ How to Run

1. Clone this repository
2. Open `analysis/baseline_survey_analysis.Rmd` in RStudio
3. Install required packages:
```r
install.packages(c("readxl", "tidyverse", "writexl", "here"))
```
4. Place data files in the `data/` folder (or use the anonymized samples already included)
5. Knit to HTML or run chunks interactively

> ⚠️ **Note:** All data has been anonymized. Respondent IDs replaced with synthetic UUIDs, ages lightly perturbed, response values perturbed ±5%, and policy question text generalized. Demographic structure and analytical relationships are preserved. See `data/DATA_DICTIONARY.md` for column definitions.
